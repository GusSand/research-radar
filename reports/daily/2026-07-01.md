## Top 10 (priority order)

1. [**Security–Fidelity Tradeoffs: The Hidden Cost of Prompt Injection Defense**](https://arxiv.org/abs/2606.30783) — Hermon, Gupta, Ruan, Sabir, Wang (Amazon / UIUC) — **ICML 2026 (accepted)** — 2026-06-29
   **Why:** An impossibility-style result that reframes how prompt-injection defense should be measured. **Summary:** Introduces **SecFid**, a benchmark separating three outcomes (execute the injection / treat it as data / ignore it) instead of one "attack success" number. Across 1,168 examples and 48 model×defense configs, no config achieves both goals — best fidelity 96.5% at only 47.8% security; most-secure defenses 99.3% security at just 71–74% fidelity. Worst for verbatim-preservation tasks (translation, doc editing).

2. [**Robust Harmful Features Under Jailbreak Attacks: Mechanistic Evidence from Attention Head Specialization**](https://arxiv.org/abs/2606.28153) — Yin, Han, Li — **ICML 2026 (Oral)** — 2026-06-26
   **Why:** Interpretability-for-security — jailbreaks *suppress* specific attention heads rather than erasing safety features; enables a training-free detector. Directly adjacent to safety-head / [[Even Heads Fix Odd Errors]] work. **Summary:** Identifies **Adversarially Compromised Heads** (early layers, suppressed during attacks) and **Safety-Aligned Heads** (mid-layers, stay active). Ablation + token-level attribution shows suppressing a few ACHs alone induces jailbreak-like compliance on normally-refused inputs; reading the persistent safety-head activations gives competitive, adversarially-robust jailbreak detection with no training.

3. [**Evaluating the Interpretability of Sparse Autoencoders with Concept Annotations**](https://arxiv.org/abs/2606.24716) — Klotz, Dantas, Jain, Marcos, Demir — **ECCV 2026 (accepted)** — 2026-06-23
   **Why:** Rigorous causal SAE-interpretability eval whose headline is that most existing SAE metrics can't even distinguish trained from random SAEs. **Summary:** Builds single-attribute-difference synthetic datasets (synCUB, synCOCO), proposes Fully-Binary Matching Pursuit for many-to-one latent→concept mapping and **TAPAScore** (tests whether matched latents respond to targeted attribute edits). Only the proposed metrics reliably separate trained from untrained SAEs; higher overcompleteness *reduces* interpretability; moderate dictionary sizes trade off best. *(3 days pre-window; strongest core-interp item found.)*

4. [**An Empirical Study of Security Calibration in Large Language Models for Code**](https://arxiv.org/abs/2606.31159) — Siddiq, Rahman, Santos (Notre Dame) — **ICSME 2026 (Research Track)** — 2026-06-30
   **Why:** Secure code generation — models are systematically overconfident about the security of code they emit, and calibration-guided repair backfires. **Summary:** Across GPT-4o-mini, Gemini-2.0-Flash, Qwen3-Coder-Next at varied temperatures, overconfidence is pervasive; functional calibration lags security calibration; calibration-guided automated repair gives limited security gains while frequently introducing functional regressions; architectural gating helped in controlled benchmarks but degraded/raised risk in realistic repo-level settings.

5. [**Improved Large Language Diffusion Models (iLLaDA)**](https://arxiv.org/abs/2606.25331) — Nie, Min, Xu, Li, Wen et al. (Renmin Univ. + ByteDance Seed) — arXiv preprint (cs.CL) — 2026-06-24
   **Why:** ◆ Text-diffusion focus — a fresh open 8B masked-diffusion base+instruct model trained from scratch; the successor reference dLLM to LLaDA. **Summary:** Keeps a fully bidirectional masked-diffusion objective through pretraining (12T tokens) + SFT (25B-token instructions, 12 epochs), adds variable-length generation and confidence-based multiple-choice scoring. Over LLaDA: +21.6 BBH, +14.9 ARC-C (base); +14.5 MATH, +16.5 HumanEval (instruct); reported competitive with Qwen2.5-7B on several benchmarks despite non-autoregressive training.

6. [**Words Speak Louder Than Code: Cognitive Heuristics in LLM-Based Code Vulnerability Detection**](https://arxiv.org/abs/2606.30587) — Shahriar, Cai, Benkraouda, Wang, Celik (Purdue / UIUC) — arXiv preprint (cs.CR) — 2026-06-29
   **Why:** LLM vuln-detectors can be manipulated purely via surrounding natural-language context, without touching the code — a real secure-code-review attack surface. **Summary:** Holds code fixed and varies only context to trigger halo (author attribution), framing (task objective), and anchoring (prior analysis) heuristics. Across 8 LLMs / 3 languages: cross-model susceptibility 33.2% (framing), 23.5% (anchoring), 18.4% (halo); a PoC cognitive attack suppressed up to 97% of previously-detected vulns; semantic-reasoning vulns more susceptible than pattern-matchable ones.

7. [**An Empirical Evaluation of Prompt Injection Vulnerabilities Across Multilingual and Obfuscated Attack Scenarios**](https://arxiv.org/abs/2606.29602) — Uysal, Birinci, Mutluergil, Çetin (Sabancı Univ.) — **AI-SS 2026 Workshop @ EDCC 2026 (peer-reviewed workshop)** — 2026-06-28
   **Why:** Documents a consistent cross-lingual safety gap — non-English prompts elicit higher malicious compliance. **Summary:** Six models (DeepSeek, GPT, Gemini, Grok, Llama, Qwen) under direct + multi-stage obfuscated injections across languages/encodings; all systematically vulnerable; non-English > English malicious compliance; DeepSeek/Gemini/Grok especially susceptible under complex instructions; character-encoding obfuscation reduces but doesn't eliminate harmful output. *(Abstract numbers qualitative.)*

8. [**Adaptive Evaluation of Out-of-Band Defenses Against Prompt Injection in LLM Agents**](https://arxiv.org/abs/2606.26479) — Narisetty, Kore, Kattamanchi, Kumarapu — arXiv preprint (cs.CR) — 2026-06-25
   **Why:** Independent adaptive-attack reproduction of five prominent out-of-band agent defenses (CaMeL, FIDES, Progent, RTBAS, FORGE). **Summary:** Adaptive (hand-crafted + optimized) attacks on AgentDojo; Progent (Qwen2.5-7B) cut mean attack success 25.8%→4.2%, best hand-crafted adaptive attack still only 2.6%. Argues deterministic out-of-band enforcement may beat in-band detection, but flags it as a single limited data point needing stronger attacks.

9. [**SemRF: A Semantic Reference Frame for Residual-Stream Dynamics in Language Models**](https://arxiv.org/abs/2606.32022) — Gu, Aleti, Chen, Zhang — arXiv preprint (cs.LG/cs.CL) — 2026-06-30
   **Why:** Residual-stream analysis attempting to separate genuine layerwise computation from "measurement drift" — a recurring depth-wise-interp confound (cf. the intervention-validity thread). **Summary:** Fixes anchor points ("Semantic Reference Frames") and measures hidden states against them via pseudo-inverse tying, Voronoi partitioning, and minimum-action "canonical traces"; relates trace complexity ("semantic knots") + an action measure to parameter efficiency. No headline benchmark numbers in the abstract — quantitative strength unverified.

10. [**Explicit Fuzzy Logic in the Feed-Forward Layer: Self-Forgetting Quantifiers Discover Legible Grammatical-Licensing Detectors**](https://arxiv.org/abs/2606.31845) — Oskin — arXiv preprint — 2026-06-30
    **Why:** Interpretable-by-construction alternative to post-hoc SAE/dictionary learning — build the FFN so each unit is a readable fuzzy-logic op. **Summary:** Replaces FFN units with explicit fuzzy set ops (intersection, set-difference) over sigmoid-bounded [0,1] memberships + soft existential/proportion quantifiers with learned forgetting rates. At 125M on OpenWebText it matches a GELU baseline while every unit stays in explicit logical form; units emerge as grammatical-licensing detectors (comparatives/passives/NPIs) predicting than/by/nor, with modest LAMBADA gains and no dictionary learning.
